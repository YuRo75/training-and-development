1 What is GitHub Copilot?

GitHub Copilot is an AI coding assistant that suggests code and entire functions in real time, answers questions about your code, and can take on multi-step development tasks. It works inside your editor and on GitHub.com.

Distinct from the GitHub manualThis manual covers the AI assistant. For Git, repositories, branches, and pull requests, see the GitHub Training Manual.

3 Agent mode

In agent mode you give a high-level task in natural language. Copilot determines the steps, edits code, runs terminal commands, invokes tools, and monitors outcomes — build results, test failures, tool output — iterating until the goal is met.

Multi-step workflows

"Connect to the dev DB, switch to AdventureWorks, then list tables with a foreign key to Customer."

Exploration at scale

"Find every stored procedure that references SalesOrderHeader and summarize each."

Delegated changes

"Add audit columns to every table in the Sales schema and regenerate the procedures."
Plan, then actUse the plan agent to create an implementation plan (saved as markdown) before agent mode makes changes — you review the approach first.

4 Where Copilot runs

  • Visual Studio and VS Code
  • JetBrains IDEs, Neovim, Eclipse, Xcode
  • SQL Server Management Studio (T-SQL help)
  • The GitHub CLI (gh copilot) and the command line
  • GitHub.com (Copilot Chat) and GitHub Mobile

7 Tips for great results

  • Write a clear comment or function signature first — Copilot completes intent.
  • Keep files open that are relevant; Copilot uses neighbouring context.
  • Break big asks into reviewable steps; prefer edit/agent mode with small diffs.
  • Always review and test AI-generated code before committing.
