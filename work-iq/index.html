Work IQ

The workplace intelligence layer for Microsoft 365 — it lets agents access and reason over how your organization actually works.

Microsoft 365Permission-awareA2A · MCP · RESTPreview

1 What is Work IQ?

Work IQ is a workplace intelligence layer that lets agents access and reason over organizational data, context, and tools. It continuously builds a semantic understanding across Microsoft 365 and external systems, with built-in, permission-aware governance.

Part of Microsoft IQWork IQ is the "how people work" pillar of Microsoft IQ, alongside Fabric IQ, Foundry IQ, and Web IQ.

2 What it captures

Documents

Files and their content across Microsoft 365.

Meetings

Recaps, decisions, and signals from meetings.

Chats

Conversations and context in Teams.

Workflows

How work moves through people and processes.
Semantic, not just searchWork IQ builds a continuously-updated semantic model of relationships between people, content, and activity — so agents reason, not just retrieve.

4 Protocols & the Work IQ API

The Work IQ API lets developers build agentic apps that securely reason over Microsoft 365 data while preserving existing permissions, compliance, and governance. Choose the protocol that fits your architecture:

ProtocolUse it for
A2A (Agent-to-Agent)Agents collaborating with other agents
MCP (Model Context Protocol)Giving an agent standardized tools/context
RESTDirect programmatic access from any app
Permissions preservedEvery protocol path enforces the user's existing Microsoft 365 permissions, compliance, and governance — access is never widened.

5 Connecting agents to Work IQ

  1. Add Work IQ MCP to your agent to extend what it can understand and do.
  2. Or call the Work IQ API (A2A / MCP / REST) from your application.
  3. Build and test from VS Code or Copilot Studio against your M365 context.
  4. Ship with governance intact — the layer enforces permissions for you.
PreviewWork IQ features marked preview are provided without an SLA and aren't recommended for production yet.
