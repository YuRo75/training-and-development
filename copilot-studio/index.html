Low-code · Agent building

Microsoft Copilot Studio

A low-code platform to build, extend, publish, and govern your own AI agents — from simple Q&A bots to autonomous, action-taking agents.

Low-codeTopics & actionsKnowledgeMulti-channel

1 What is Copilot Studio?

Microsoft Copilot Studio is the graphical, low-code tool for building and customizing agents. Use it to create standalone agents or to extend Microsoft 365 Copilot, then publish to the channels where your users already are.

Part of the Power PlatformIt sits alongside Power Apps, Power Automate, and Dataverse, so agents reuse your connectors, flows, and data.

2 Core concepts

TermMeaning
AgentThe assistant you build — its persona, knowledge, and skills.

ActionA task the agent performs — a connector, Power Automate flow, API, or MCP tool.

3 Two ways to author: generative vs. classic

Modern agents are mostly generative: you give instructions and knowledge, and the model orchestrates responses dynamically. Add explicit topics only where you need deterministic control.

StyleHow it behavesUse when
Generative (AI-based)Model plans responses from instructions + knowledgeMost agents — flexible, fast to build
Classic (topics)You design exact trigger-to-node conversation flowsCompliance scripts, fixed flows, hand-offs
Start generativeBegin with instructions + a knowledge source. Add topics surgically where you must guarantee a specific path.

4 The build lifecycle

  1. Build — describe the agent in natural language; add instructions, knowledge, actions.
  2. Test — chat in the built-in test pane; watch how it reasons.
  3. Publish — release to one or more channels.
  4. Analyze — review usage, resolution rate, and transcripts.
  5. Improve — refine instructions, topics, and knowledge from real conversations.

5 Build your first agent

  1. Go to copilotstudio.microsoft.com and select Create → New agent.
  2. Describe its purpose in plain language; Copilot Studio drafts a name, instructions, and starter topics.
  3. Add a knowledge source — a SharePoint site, a public website, or uploaded files.
  4. Open the Test pane and chat; refine the instructions until answers are right.
  5. Select Publish, then turn on a channel such as Teams or a website.
Knowledge + actions = a useful agentKnowledge makes answers accurate; actions let the agent do things (look up an order, file a ticket).

6 Knowledge & actions

Knowledge

Ground the agent in SharePoint, public websites, Dataverse tables, uploaded files, or enterprise data so answers stay accurate and current.

Actions

Let the agent act: 1,000+ connectors, Power Automate flows, REST APIs, or MCP tools.
MCP supportCopilot Studio can consume Model Context Protocol servers, giving agents standardized external tools and data sources.

7 Publish & channels

One agent, many surfaces:

  • Microsoft Teams and Microsoft 365 Copilot
  • A website or custom app (web chat)
  • Custom channels via Direct Line / mobile
  • Voice and telephony scenarios
Match the authenticationPick the right auth (no auth, Entra ID, or manual) per channel so the agent only exposes data the end user may see.
