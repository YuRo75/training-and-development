1 What is Microsoft Azure?

Microsoft Azure is a cloud computing platform with a broad catalog of services you rent on demand — from virtual machines to managed databases and AI. You pay for what you use instead of buying and running your own hardware.

ModelYou manageExample
IaaSOS, runtime, appVirtual Machines, Virtual Network
PaaSJust your app/dataApp Service, Azure SQL Database
SaaSNothing (just use it)Microsoft 365, Dynamics 365

2 Global infrastructure

The Microsoft Cloud spans 200+ datacenters organized into a hierarchy that gives you choice over residency, latency, and resilience:

ConceptMeaning
GeographyA data-residency boundary (e.g. Europe, United States)
RegionA set of datacenters in one area — 70+ worldwide
Availability ZonePhysically separate locations within a region
Region pairTwo regions for cross-region disaster recovery
Pick regions deliberatelyRegion choice affects compliance, latency, cost, and which services are available. Check data residency first, then proximity to users.

4 Hands-on with the Azure CLI

The Azure CLI (az) creates and manages resources from the command line. The everyday loop is: log in → create a resource group → create resources inside it.

# 1. Sign in
az login

# 2. A resource group holds related resources
az group create --name rg-demo --location westeurope

# 3a. A Linux VM
az vm create --resource-group rg-demo --name myVM \
  --image Ubuntu2204 --admin-username azureuser --generate-ssh-keys

# 3b. A storage account
az storage account create --name mystore123 --resource-group rg-demo \
  --location westeurope --sku Standard_LRS

# 3c. Deploy a web app straight from the current folder
az webapp up --name my-demo-app --runtime "NODE:20-lts"
Clean up to stop billingDelete everything in one shot: az group delete --name rg-demo --yes. Resource groups make teardown trivial.

7 Get started

  1. Create an Azure account (the free tier has credits + always-free services).
  2. Install the Azure CLI and run az login.
  3. Create a resource group and deploy something small (a web app or storage account).
  4. Add a budget alert and tags before you scale up.
