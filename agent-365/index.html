Control plane for AI agents

Microsoft Agent 365

Microsoft's control plane for AI agents — a central way to register, observe, govern, and secure every agent across your organization.

RegistryGovernSecureObserve

1 What is Agent 365?

Microsoft Agent 365 is Microsoft's control plane for AI agents. As organizations adopt agents across Microsoft platforms, partner ecosystems, and custom apps, Agent 365 gives a centralized way to observe, govern, and secure them all.

Think "management plane for agents"Just as you manage users and devices centrally, Agent 365 is where you manage agents — whoever built them and wherever they run.

2 Why it exists

Agents are multiplying fast — built in Copilot Studio, Azure AI Foundry, by partners, and in-house. Without a control plane you get agent sprawl: unknown agents, unclear access, no consistent oversight.

Visibility

Know which agents exist and what they can do.

Control

Apply consistent policy and access across all of them.

Trust

Secure and audit agents like any other enterprise identity.

3 What Agent 365 provides

PillarWhat it does
RegistryA central inventory of agents across platforms and ecosystems
Identity & accessAgents get governed identities (Entra) and scoped permissions
SecurityProtect and monitor agents with Microsoft Defender
GovernanceApply data and compliance controls with Microsoft Purview
ContextGround agents in organizational context via Work IQ
InteropWork with agents from Microsoft, partners, and custom builds

4 The Agent Registry

The Agent Registry is converging into Agent 365 as the single inventory of your agents. It's the foundation for observing, governing, and securing agents at scale.

One registry, many sourcesAgents built in different tools register centrally — so policy, security, and audit apply uniformly rather than tool-by-tool.

5 Managing the agent lifecycle

  1. Onboard — register the agent and assign it a governed identity.
  2. Secure — scope its access and bring it under Defender/Purview.
  3. Observe — monitor activity, usage, and risk centrally.
  4. Retire — decommission agents cleanly when no longer needed.
Emerging platformAgent 365 is new and evolving. Treat capabilities as a direction of travel and confirm current availability in Microsoft Learn before relying on specifics.
